## Firestore Queries

### Query 1: Get User Document

**Dart Code:**
```dart
final userDoc = await FirebaseFirestore.instance
    .collection('tbl_users')
    .doc('xq8i7ChN5ycgnSdu7V6PLpHwkmT2')
    .get();

if (userDoc.exists) {
  final userData = userDoc.data();
  print('User found: $userData');
} else {
  print('User not found');
}
```

**Firebase Console:** 
- Navigate to `tbl_users` collection
- Search for document ID: `xq8i7ChN5ycgnSdu7V6PLpHwkmT2`

---

### Query 2: Get All Posts by User

**Dart Code:**
```dart
final postsSnapshot = await FirebaseFirestore.instance
    .collection('tbl_posts')
    .where('user_id', isEqualTo: 'xq8i7ChN5ycgnSdu7V6PLpHwkmT2')
    .orderBy('timestamp', descending: true)
    .get();

print('Found ${postsSnapshot.docs.length} posts');

for (var doc in postsSnapshot.docs) {
  final post = doc.data();
  print('Post: ${post['content']} (${post['timestamp']})');
}
```

**Firebase Console:**
- Navigate to `tbl_posts` collection
- Click "Add filter"
- Set: `user_id` → `==` → `xq8i7ChN5ycgnSdu7V6PLpHwkmT2`

## Troubleshooting

### Issue: "User not found"
**Possible Causes:**
- Wrong user ID (check for typos, case sensitivity)
- User exists in Firebase Auth but not in Firestore
- Firestore rules blocking access

**Solution:**
```dart
// List all users to verify the ID exists
final allUsers = await FirebaseFirestore.instance
    .collection('tbl_users')
    .limit(10)
    .get();

for (var doc in allUsers.docs) {
  print('User ID: ${doc.id}');
}
```

### Issue: "No posts found"
**Possible Causes:**
- User hasn't created any posts
- `user_id` field spelling doesn't match exactly
- Firestore rules blocking read access

**Solution:**
```dart
// Check first few posts to see structure
final firstPosts = await FirebaseFirestore.instance
    .collection('tbl_posts')
    .limit(3)
    .get();

for (var doc in firstPosts.docs) {
  print('Post user_id: ${doc['user_id']}');
}
```

### Issue: "Firestore rules not found"
- Rules are managed in Firebase Console
- Navigate to: https://console.firebase.google.com/project/microblogging-app-121c9/firestore/rules
- May be configured to allow public read (test mode)

## Firestore Rules Configuration

To view/edit Firestore security rules:

1. Go to: https://console.firebase.google.com/project/microblogging-app-121c9/firestore/rules
2. Common rule patterns used in social apps:

```firestore
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow users to read their own data
    match /tbl_users/{userId} {
      allow read: if request.auth.uid == userId;
      allow write: if request.auth.uid == userId;
    }
    
    // Allow public read of posts, auth required for write
    match /tbl_posts/{postId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```
